## §2 Per-category root cause

### Category A — `cargo test --workspace` writes to hard-coded old evidence dirs

**Smoking-gun evidence**: each TB-7 / TB-13 / TB-14 cargo test contains an unconditional best-effort write block to a hard-coded May-01/May-03 path. When `cargo test --workspace` re-runs on a newer kernel (different keystore RNG seed → different ed25519 pubkeys, different state-root hashing of new tx fields), it overwrites:

| File | Writer | Line | Hard-coded path |
|------|--------|------|------------------|
| `tb_7_chaintape_smoke_2026-05-01/{replay_report,chain_derived_run_facts,agent_pubkeys,README}.json/md` | `tests/tb_7_atom6_chain_backed_smoke.rs` | 243-254 | `Path::new("handover/evidence/tb_7_chaintape_smoke_2026-05-01")` |
| `tb_13_chaintape_smoke_2026-05-03/{replay_report,agent_pubkeys,README}.json/md` | `tests/tb_13_chaintape_smoke.rs` | 424-446 | `Path::new("handover/evidence/tb_13_chaintape_smoke_2026-05-03")` |
| `tb_14_chaintape_smoke_2026-05-03/{replay_report,agent_pubkeys,pinned_pubkeys,genesis_report,README}.json/md` | `tests/tb_14_chaintape_smoke.rs` | 351-381 | `Path::new("handover/evidence/tb_14_chaintape_smoke_2026-05-03")` |

The path is a literal `Path::new(...)` with **no env override, no timestamp suffix, no exists()-check guard**. The comment on tb_7 line 241 even acknowledges "best-effort; if dir unwritable, the test still passes" — i.e., **writes-by-default** semantics. The pubkey diff (different SHA-256 in agent_pubkeys.json) and changed `final_state_root_hex` / `head_commit_oid_hex` are perfectly consistent with re-deriving keystores from a fresh tempdir on the running kernel — keystore generation in cargo tests typically uses `tempfile::TempDir` so each run produces fresh keys, hence different pubkeys → different signatures → different state-root tx-chain.

For tb_14 the `genesis_report.json` diff `+ "agent_autopsies_t": {}` is a schema bump (new QState field added in TB-18R / TB-C0 post-2026-05-03), confirming the file was regenerated by a newer kernel. The tb_7 `chain_derived_run_facts.json` got 7 new fields (`expected_completed_attempts`, `l4_work_attempt_count`, `l4e_work_attempt_count`, `attempt_aborted_count`, `capsule_anchored_attempt_count`, `delta`, `terminal_halt_class`) — these are TB-18R FR-18R.3 / TB-C0 invariant additions to `compute_run_facts_from_chain`, again confirming newer-kernel regeneration.

## §3 Recommendations (informs A1 evidence-immutability hook)

The runner-preflight skill's stage 3 (`git diff --stat HEAD -- handover/evidence/`) catches drift **before** running a runner — but does NOT catch drift caused by `cargo test --workspace` (which has no preflight). Required hook design:

1. **Pre-commit hook stage** that rejects any staged change to `handover/evidence/<TB-name>_*-<YYYY-MM-DD>/` paths older than e.g. 7 days, unless explicit override env (`EVIDENCE_REWRITE_AUTHORIZED=<reason>`). Add to `scripts/hooks/pre-commit.r022` shim or new sibling `pre-commit.evidence-immutable`.
2. **Cargo-test-time guard**: convert hard-coded `handover/evidence/tb_*` paths in `tests/tb_{7,13,14}*.rs` to:
   - opt-in via env (`TURINGOS_PERSIST_EVIDENCE_DIR=...`), default = NO write to repo paths; OR
   - timestamp suffix (e.g., `tb_7_chaintape_smoke_$(date +%Y-%m-%d)/<commit-shortsha>/`); OR
   - keep tmp-only and require explicit script (`scripts/regenerate_tb7_smoke.sh`) for canonical dump.
3. **Repo-wide grep gate**: CI test that rejects `Path::new("handover/evidence/")` in `tests/**/*.rs` (write-side-effects from cargo test forbidden).
