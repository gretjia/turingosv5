Author: Claude (researcher). External audit pending: Codex + Gemini.

## Prelude — Why v3

V2 的失败暴露了三层问题（经双审计确认）：
1. **叙述错误**：n3 的 "+33% PPUT" 实为 "k=3 sampling advantage"（agents 从不 `append`，swarm 机制未激活）。
2. **测量框架破碎**：指标层级、样本抽取、角色分离、熔断规则全部缺失。
3. **hybrid_v1 预算被瓜分**，数据报废。

V3 的目标 ≠ 再争取 PPUT 分数，而是**重建正确的测量基座**，在此基础上做**因果可归因的架构实验**。

## Part 3: Experiments E1 + E2（同批并跑）

### E1：独立样本重复性验证
**问题**：在干净（非我挑选）的 50 题上，oneshot vs n3 是否仍显 n3 有 +OMEGA 增益？

**对齐**：C-010（样本外部）、C-012（冻结指标）

### E2：n1 机制隔离
**问题**：若 n3 领先 oneshot，领先来自"swarm 机制"还是"3 次采样"？

**对齐**：C-033（因果可归因）、Art. II.2.1（探索/利用）、C-034（机制优于 prompt）

### 合并批次设计
50 题 × 3 条件（oneshot / n1 / n3）= 150 runs.

**推理**：
- oneshot vs n1 = 隔离 swarm 机械税（单 agent 跑完整 machinery）
- n1 vs n3 = 隔离多样本收益
- oneshot vs n3 = 复现 v2 主结论
- 三者同样本、同预算（900s per problem）、同时间窗运行（控制 API 漂移）

**预算**：150 × ~150s avg = ~6.25h。熔断可能把坏条件砍到 ~20-30 题即停，实际 ~4-7h。

### 判据预注册
| 比较 | 可能结论 |
|---|---|
| oneshot ≈ n1 (OMEGA%) | swarm 机制零税零益 → 机制实际上无功能 |
| oneshot > n1 | swarm 机制**纯税** → 必须重设计（E3 方向） |
| n1 ≈ n3 | 多样本无收益 → "n3 比 oneshot 好" 完全来自机制（if n1>oneshot） |
| n3 > n1 > oneshot | 机制 + 采样都有益 |
| n3 > n1 = oneshot | 纯采样收益 → v2 叙述确认 |
