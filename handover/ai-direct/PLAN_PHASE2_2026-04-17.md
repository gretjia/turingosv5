## 4-Step Plan (each = N=20 A/B, ~3h, ~$12)

### P3. Hybrid Mode (immediate, 30 min implement)
- n=1: direct-complete allowed (no force-append)
- n>1: force-append preserved (multi-agent needs tape for coordination)
- Constitutional: Art. III.3 (correlation shielding meaningless at n=1)
- Expected: n1 recovers to 55%+, n3 retains tape benefits
- File: evaluator.rs only (non-restricted)

### P1. Diff-Only Append (root cause fix)
- Redefine append: agent writes ONLY the new tactic delta, not full proof
- Scaffold assembles full proof from chain (root → parent → new delta)
- Auto-probe tests the ASSEMBLED chain, not individual delta
- Constitutional: Art. III.2 (encapsulate details; scaffold owns assembly)
- Expected: payload −80%, probe accuracy up, tape depth meaningful
- Files: prompt.rs + evaluator.rs

### P2. Conditional Probe (smart scheduling)
- Probe only when latest error class differs from previous (strategy changed)
- Skip probe when agents repeat same failing tactic class
- Uses existing classify_lean_error infrastructure
- Constitutional: Art. I.1 (deterministic oracle → same input same output → no repeat probe)
- Expected: oracle calls −40-60%
- File: evaluator.rs

### P4. Chain-Prefix Deduplication (context hygiene)
- Suppress near-duplicate ancestors in agent prompt
- Show only assembled chain + error delta
- Constitutional: Art. III.2 (context is zero-sum; inject less = more attention per token)
- Expected: prompt tokens −70%, output quality up
- File: prompt.rs

## Execution Protocol

1. P3 → N=20 A/B → if n1 recovers ≥ 55%: proceed

- n3 > n1 by ≥ 3 on N=50 (STRICT WIN = super-linear signal)
- Tape depth ≥ 5 (architecture alive)
- Oracle calls per problem < 10 (overhead controlled)
