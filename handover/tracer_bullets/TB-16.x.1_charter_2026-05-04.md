## §1 Background

OBS_TB_16_TAMPER_R2_HANG (`handover/alignment/OBS_TB_16_TAMPER_R2_HANG_2026-05-04.md`)
documents that `audit_tape_tamper` Round 2 (`flip_cas_byte` corruption)
hangs indefinitely on the `audit_pipeline_smoke/` fixture's MarkovEvidenceCapsule
loose object. Round 2 8/8 detect on richer chains confirms it is fixture-state-
specific: the regenerated capsule's specific byte pattern, when back-half-zeroed,
triggers an unbounded-allocation / infinite-loop path in some downstream
assertion that does NOT route through `CasStore::get`'s sha256 verify gate.

R3 ship gate accepted PROCEED on audit_tape with the tamper anomaly OBS-
deferred. TB-17 RealWorld Gate is Class 4 (constitution sudo); audit
pipeline DoS-resistance is a TB-17 prerequisite, hence root-causing here.

## §2 Problem statement

```text
Given:
  CAS loose object whose back half is zeroed (flip_cas_byte semantic).
  CasStore::get sha256-verifies and rejects with CidMismatch fast.

Find:
  The audit pipeline path that reads adversarial CAS bytes WITHOUT routing
  through CasStore::get, then feeds them to canonical_decode (bincode), which
  on a length-prefixed type can attempt to allocate ~2^60 bytes or loop on
  garbage.

Fix at the lowest meaningful boundary:
  Either (a) route the offending assertion through CasStore::get; or
        (b) bound canonical_decode buffer size at CAS-get layer per
            metadata.size_bytes (defense-in-depth).
  Probably both.
```

### Atom C — Surgical fix at CAS-get + assertion layer (P1.B, ~2-3 hr)

Two changes (both, defense-in-depth):

**C.1 — `CasStore::get` adversarial-bytes bound**:
- After `find_blob` returns content, check `content.len() <= metadata.size_bytes` BEFORE sha256 verify.
- If exceeded: return `CasError::SizeExceedsIndex { expected, actual }` (new variant).
- If sha256 mismatch (current behavior): return `CasError::CidMismatch` (unchanged).
- This bounds the buffer that downstream `canonical_decode` sees, even if the assertion bypasses sha256 (it shouldn't, but defense-in-depth).

**C.2 — Identified-assertion routing**:
- Whichever assertion id Atom B identifies: route its CAS read through `CasStore::get`
  if it currently uses `repo.find_blob` directly.
- If it MUST read raw (e.g., walking the entire ODB), wrap the
  `canonical_decode::<T>(&bytes)` call with a `bincode::DefaultOptions::with_limit(metadata.size_bytes)`
  bound or equivalent.

**C.3 — Test**: `tests/cas_adversarial_bytes.rs` new file.
- t1: `CasStore::get` rejects an oversized blob with `SizeExceedsIndex`.
- t2: `CasStore::get` rejects a corrupted-content blob with `CidMismatch` (regression-guard for current behavior).
- t3: `audit_tape_tamper` on a reduced-fixture analog completes within 5s.
