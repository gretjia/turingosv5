### Atom 3 — EvidenceCapsule writer + CAS integration (Class 1)

**Action**: new module `src/runtime/evidence_capsule.rs`.

API:
```rust
pub fn write_evidence_capsule(
    cas: &CasStore,
    run_id: RunId,
    task_id: TaskId,
    solver_agent: Option<AgentId>,
    counts: ExhaustionCounts,
    rounds: (u64, u64),
    terminal_reason: ExhaustionReason,
    raw_log_bytes: &[u8],
    privacy: CapsulePrivacyPolicy,
) -> Result<EvidenceCapsule, CapsuleWriteError>;
```

Internal flow:
1. Compress `raw_log_bytes` via flate2/gzip → `compressed_log_bytes`.
2. Write `compressed_log_bytes` to CAS → returns `compressed_log_cid: Cid`.
3. Build manifest JSON: `{ "compressed_log_cid": ..., "size_bytes": ..., "sha256_pre_compress": ..., "sha256_post_compress": ... }`.
4. Write manifest to CAS → returns `evidence_manifest_cid: Cid`.
5. Build `public_summary: String` from counts (e.g. `"132 attempts; 73 lean errors; 14 sorry-blocks; 26 parse failures; no accepted proof"`) — this is the only field surfaced to dashboard / agent read views.
