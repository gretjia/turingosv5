## §0 Why TB-15 exists (architect §6 — verbatim)

```text
TB-15 — Lamarckian Autopsy + Markov EvidenceCapsule

把失败、爆仓、亏损、反复错误转化为私有学习与 Markov capsule，
而不是全局 raw-log 污染。
```

## §1.1 Privacy contract (architect §6.4 — binding)

```text
Default privacy_policy = CapsulePrivacyPolicy::AuditOnly (re-use TB-11).

  - public_summary: low-info string (loss_reason_class + amount + event_id);
                    eligible for typical-error broadcast IF clustering hits.
  - private_detail_cid: CAS Cid of detailed analysis JSON; readable ONLY
                        through audit-role surface; NEVER enters
                        AgentVisibleProjection or any agent prompt.
  - evidence_cids: Vec<Cid> pointing to ChainTape sub-evidence (loss
                   tx, slash tx, position state, market pool state).
                   Cids are public-Cid-of-existing-public-evidence; not
                   raw new bytes.

Forbidden (architect §6.6 verbatim):
  No global raw autopsy broadcast.
  No forced prompt stuffing of all past failures.
  No automatic predicate mutation.
  No MetaTape self-modification.
  No constitution change.
  No hidden source-of-truth capsule.
  No private loss detail in public read view.
```

## §1.3 Markov default contract (architect §6.3 FR-15.4 + FR-15.5 — binding)

```text
FR-15.4: "Next InitAI context defaults to constitution + latest capsule."
FR-15.5: "Markov override is required for deep history reads."

TB-15 v0 implementation:
  - generate_markov_capsule binary writes capsule to:
      CAS as ObjectType::MarkovEvidenceCapsule
      handover/markov_capsules/MARKOV_TB-N_<date>.json (pointer file)
      handover/markov_capsules/LATEST_MARKOV_CAPSULE.txt (Cid hex pointer)
  - LATEST_MARKOV_CAPSULE.txt = current Markov head; updated on each
    capsule generation.
  - next_session_context_cid → CAS Cid of a JSON blob containing:
      { "constitution_hash": ..., "latest_markov_cid": ..., "boot_seq":
        ["read constitution.md", "read CAS<latest_markov_cid>"] }
  - The default-deny mechanism is binary-level:
      generate_markov_capsule reads constitution + previous Markov
        capsule by default.
      If TURINGOS_MARKOV_OVERRIDE=1 is set, additional history
        sources can be ingested.
      Otherwise, attempting to read deeper history (older capsules,
        L4 chain rows pre-dating the previous Markov capsule's
        l4_root) returns DeepHistoryReadDenied.

InitAI / agent-side enforcement is OUT OF SCOPE for TB-15 (no InitAI
agent shipped yet — P5 v1 territory). TB-15 ships the substrate +
default-deny semantics in the generator binary; agent-side honoring
of the Markov default is documented as a P5 follow-up requirement.
```
