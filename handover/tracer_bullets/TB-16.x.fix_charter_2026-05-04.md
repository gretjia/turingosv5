### Atom 2 — `AuditInputs.markov_pointer: Option<PathBuf>` + binary CLI changes

- Change `AuditInputs.markov_pointer: PathBuf` → `Option<PathBuf>`.
- `load_tape` body: replace `if inputs.markov_pointer.exists() { … } else { None }` with `match &inputs.markov_pointer { Some(p) if p.exists() => Some(read_markov_capsule(p, &cas)?), Some(p) => return Err(AuditError::MarkovPointerUnresolvable(format!("{:?}", p))), None => None }`. **Refinement**: per architect Q2.c, the CLI must distinguish "genesis" from "passed-but-not-found"; absent CLI flag → `None`; present-but-FS-absent → BLOCK (not silent None).

  Wait — rethink: `load_tape` currently uses `pointer.exists()` to branch. The architect ruling says:
  - `pointer absent (CLI not passed)` → `markov_capsule = None` → genesis (correct).
  - `pointer present-but-FS-absent` → currently silently `None`; this is the bug the architect calls out indirectly via "pointer absent → None is correct".
  - `pointer present + FS-present + CAS-cannot-resolve` → already BLOCKs post-TB-16.x.1.

  Decision: with `Option<PathBuf>`, `None == genesis`; `Some(p)` requires `p.exists()` AND `read_markov_capsule(p, &cas)` to succeed; otherwise BLOCK.
- `audit_tape` binary: drop `markov_pointer.ok_or("--markov-pointer required")?`; replace with `markov_pointer: markov_pointer` (Option<PathBuf>); add `--prior-chain-runtime-repo` flag; if both flags absent → genesis; if `--prior-chain-runtime-repo` present → resolve to a `markov_pointer` path inside the prior chain's runtime_repo (e.g. read `<prior>/.markov_head` or equivalent).
- `audit_tape_tamper`: mirror.

**`--prior-chain-runtime-repo` resolver semantic**: the prior chain's tip Markov capsule cid must be discoverable from the prior chain's `runtime_repo` itself (in-tape). For α immediate, the resolver is permitted to be a thin wrapper that reads a per-run `MARKOV_TB-*.json` from a known per-run subdir (NOT the global pointer). The full Art. 0.4 path B in-tape encoding lands in TB-16.x.2.4.
