## 1. Verdict
Partial confirmation, but the requester's framing is too narrow. `Node.completion_tokens` being zero is real, and `gp_token_count` is byte-length, not token accounting. However, adding a `completion_tokens` parameter to `bus.append` would not satisfy the tape-canonical axiom: failed LLM calls, rejected proofs, market/wallet state, rejection feedback, search hits, board/memory files, FC trace, and wall-clock timing all affect agents, settlement, or reported metrics while living outside `bus.kernel.tape`.

## 2. Confirmed violations
- `Node` has canonical-looking fields `created_at` and `completion_tokens` (`src/ledger.rs:18-24`), but production append constructs the node with `completion_tokens: 0` (`src/bus.rs:259-268`). Real API token counts exist in `GenerateResponse` (`src/drivers/llm_http.rs:31-37`) and are parsed from proxy `usage` (`src/drivers/llm_http.rs:131-145`).
- Real C_i accounting is a parallel run ledger: `RunCostAccumulator` stores prompt/completion/tool tokens and proposal/failure counts (`experiments/minif2f_v4/src/cost_aggregator.rs:29-38`), records calls (`experiments/minif2f_v4/src/cost_aggregator.rs:49-60`), and computes `total_run_token_count` (`experiments/minif2f_v4/src/cost_aggregator.rs:95-98`). It is fed before parse/accept is known (`experiments/minif2f_v4/src/bin/evaluator.rs:1037-1040`).
- `gp_token_count` is not tokens. The swarm success paths sum `n.payload.len()` over tape nodes and max with one response's completion tokens (`experiments/minif2f_v4/src/bin/evaluator.rs:1211-1217`, `experiments/minif2f_v4/src/bin/evaluator.rs:1461-1465`). `make_pput` then emits that as both `golden_path_token_count` and legacy `gp_token_count` (`experiments/minif2f_v4/src/bin/evaluator.rs:1705`, `experiments/minif2f_v4/src/bin/evaluator.rs:1734`).

## 3. Additional violations found
- Economy state is canonical but not tape-reconstructible. `Kernel` stores `markets`, `bounty_market`, and LP seed beside tape (`src/kernel.rs:19-31`); prices are snapshotted into agent prompts (`src/bus.rs:541-579`, `experiments/minif2f_v4/src/bin/evaluator.rs:1001-1003`). Direct evaluator `invest` mutates wallet and kernel market books without appending a tape node or even using the bus invest ledger path (`experiments/minif2f_v4/src/bin/evaluator.rs:1291-1345`). Wallet balances/portfolios live in `WalletTool` maps (`src/sdk/tools/wallet.rs:24-28`, `src/sdk/tools/wallet.rs:45-77`) and can persist cross-problem via `WALLET_STATE` (`experiments/minif2f_v4/src/bin/evaluator.rs:617-630`, `src/sdk/tools/wallet.rs:83-97`).
- Rejection feedback is an off-tape prompt signal. `TuringBus` stores a `graveyard` (`src/bus.rs:48`), writes class labels in `record_rejection` (`src/bus.rs:444-449`), derives Top-K feedback (`src/bus.rs:519-536`), and the evaluator injects it into prompts (`experiments/minif2f_v4/src/bin/evaluator.rs:940`, `experiments/minif2f_v4/src/bin/evaluator.rs:1001-1003`).
- Search and Librarian state are off-tape prompt signals. Search hits are cached in `search_cache` (`experiments/minif2f_v4/src/bin/evaluator.rs:758-759`), injected into prompts (`experiments/minif2f_v4/src/bin/evaluator.rs:980-1003`), and updated from filesystem search (`experiments/minif2f_v4/src/bin/evaluator.rs:1350-1374`, `src/sdk/tools/search.rs:21-50`). Learned memory and board posts are read/written as files (`src/sdk/tools/librarian.rs:61-118`) and injected into prompts (`experiments/minif2f_v4/src/bin/evaluator.rs:953-1003`, `experiments/minif2f_v4/src/bin/evaluator.rs:1378-1389`).
- FC trace is a separate append-only event stream with `ts_ms`, `fc_id`, `run_id`, `tx`, and arbitrary kv (`experiments/minif2f_v4/src/fc_trace.rs:21-30`, `experiments/minif2f_v4/src/fc_trace.rs:127-177`). It records mr ticks and halt events off-tape (`experiments/minif2f_v4/src/bin/evaluator.rs:850-857`, `experiments/minif2f_v4/src/bin/evaluator.rs:1228-1236`, `experiments/minif2f_v4/src/bin/evaluator.rs:1585-1594`).

## 4. Stress-test results
a. Confirmed: `make_pput` uses accumulator C_i, not `gp_tokens`, for inferential math. Success/no-success call sites pass `acc.total_run_token_count()` into `make_pput` (`experiments/minif2f_v4/src/bin/evaluator.rs:1266`, `experiments/minif2f_v4/src/bin/evaluator.rs:1499`, `experiments/minif2f_v4/src/bin/evaluator.rs:1602`). Inside `make_pput`, `c_i = total_run_token_count`, `t_i = total_wall_time_ms`, and both `pput_runtime` and `pput_verified` call `compute_pput(..., c_i, t_i)` (`experiments/minif2f_v4/src/bin/evaluator.rs:1663-1672`). `gp_tokens` only populates output fields (`experiments/minif2f_v4/src/bin/evaluator.rs:1705`, `experiments/minif2f_v4/src/bin/evaluator.rs:1734`). H1-H4 math is preserved on this axis.

b. Yes: there are multiple second ledgers. Besides `RunCostAccumulator`, the proxy has global token `/stats` counters (`src/drivers/llm_proxy.py:128-146`, `src/drivers/llm_proxy.py:271-278`); kernel market books and wallet portfolios mutate off-tape (`src/kernel.rs:129-142`, `src/sdk/tools/wallet.rs:45-77`, `experiments/minif2f_v4/src/bin/evaluator.rs:1312-1335`); FC trace is its own event log (`experiments/minif2f_v4/src/fc_trace.rs:127-177`).

c. No: `Node.created_at` is seconds-resolution per appended node (`src/bus.rs:264-268`). Phase E reproducibility for PPUT would need first-read/final-accept and verifier intervals, which live in `RunWallClock` using `Instant` (`experiments/minif2f_v4/src/wall_clock.rs:51-74`) and evaluator call brackets (`experiments/minif2f_v4/src/bin/evaluator.rs:828-834`, `experiments/minif2f_v4/src/bin/evaluator.rs:1119-1174`, `experiments/minif2f_v4/src/bin/evaluator.rs:1423-1446`). For node ordering, `time_arrow` is enough; for wall-clock cost, `created_at` is insufficient.

## 6. Hidden risks
- Blast radius is larger than `completion_tokens`. `bus.append` has 13 occurrences: one production call (`experiments/minif2f_v4/src/bin/evaluator.rs:1058`) and 12 bus tests (`src/bus.rs:610-736`). `append_oracle_accepted` has three production call sites (`experiments/minif2f_v4/src/bin/evaluator.rs:1201`, `1458`, `1510`).
- Existing `Node` fixtures will break if new fields lack serde/default constructors: `src/ledger.rs:349-355`, `src/kernel.rs:249-255`, `src/sdk/actor.rs:175-181`, `src/sdk/actor.rs:249-255`, `src/wal.rs:112-118`.
- `completion_tokens = 0` is mostly inert today: searches found assignments but no production reads of `Node.completion_tokens`; switching `gp_token_count` to read it will make old WAL/tape rows report zero unless migrated/defaulted.
- Adding per-node tokens does not capture failed branches. Tokens are recorded before parse/accept (`experiments/minif2f_v4/src/bin/evaluator.rs:1037-1040`), while parse fails and Lean rejects may not append nodes (`experiments/minif2f_v4/src/bin/evaluator.rs:1275-1281`, `experiments/minif2f_v4/src/bin/evaluator.rs:1541-1547`).

## 7. Independent challenge
The axiom needs a sharper boundary. If it literally means every signal must be reconstructible from `Vec<Node>`, this refactor is architectural, not a token-field patch. If it means “proof-state tape only,” then markets, wallet, FC trace, and wall-clock should be declared noncanonical side telemetry instead of being half-canonical. The dangerous refactor is to pass `completion_tokens` into `append`, declare victory, and leave failed-branch cost, investments, rejection feedback, search cache, and wall-clock off-tape.

Bottom line: PROCEED_WITH_MODIFICATIONS.
