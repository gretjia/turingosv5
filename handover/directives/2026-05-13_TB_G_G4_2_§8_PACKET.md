## §1. Intent

G4's goal is not merely to run several API model names. The goal is to make
per-agent model identity a replayable constitutional fact:

```text
Agent_i -> genesis-assigned model identity -> AttemptTelemetry actual model
         -> audit assertion: no hidden model switch
         -> dashboard/report divergence by model family
```

Existing code already has an env resolver:

- `experiments/minif2f_v4/src/agent_models.rs`
- `AGENT_MODELS`
- `PHASE_D_HETERO_OK=1`

That resolver is currently startup/runtime configuration. G4.2 promotes the
resolved assignment into genesis evidence so replay and audit can answer:

1. Which model was each agent assigned at batch genesis?
2. Which model actually produced each attempt?
3. Did any attempt use a different model without a ChainTape/CAS-visible
   record?

## §3. Proposed Data Contract

### §3.1 Genesis assignment shape

Add a deterministic, serde-stable field to `GenesisReport`:

```rust
pub agent_model_assignment: Vec<AgentModelAssignment>,
```

Recommended record shape:

```rust
pub struct AgentModelAssignment {
    pub agent_id: String,
    pub model_name: String,
    pub model_family: String,
    pub provider: String,
    pub model_version: Option<String>,
    pub temperature_milli: i64,
    pub prompt_template_hash: Option<String>,
}
```

Rules:

- Deterministic order: sorted by `agent_id`.
- Missing legacy field: `#[serde(default)]` on the vector so historical
  `genesis_report.json` remains parseable.
- `temperature_milli` is integer-only. No `f64` / `f32`.
- `prompt_template_hash` is a hash/id only. Do not store raw prompt bodies or
  private prompt-completion traces.
- `model_family` is a normalized coarse class such as `claude`, `openai`,
  `qwen`, `deepseek`, `local`, or `unknown`.

### §3.2 AttemptTelemetry actual-model witness

Each new AttemptTelemetry emitted after G4.2 must carry the actual model used
for that LLM call. The preferred approach is tail-additive fields on
`AttemptTelemetry`:

```rust
#[serde(default)]
pub model_name: Option<String>,
#[serde(default)]
pub model_family: Option<String>,
#[serde(default)]
pub model_provider: Option<String>,
#[serde(default)]
pub model_version: Option<String>,
#[serde(default)]
pub temperature_milli: Option<i64>,
```

Implementation must prove one of these:

- old AttemptTelemetry bytes still decode with the tail-additive fields, or
- the schema is bumped with a dual-reader / grandfathering path and tests prove
  old evidence is not retroactively rewritten or made undecodable.

### §3.3 No hidden switch semantics

Default rule:

```text
AttemptTelemetry.actual_model(agent_id, attempt_id)
==
GenesisReport.agent_model_assignment[agent_id]
```

If a future design permits explicit model switching, that switch must be a
separate ChainTape/CAS-visible event with its own §8 packet. G4.2 does **not**
authorize model switching. For this atom, mismatches fail the no-hidden-switch
assertion.
