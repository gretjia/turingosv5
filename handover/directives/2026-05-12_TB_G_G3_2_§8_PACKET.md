### Q1 — Risk-cap value: per-agent-keyed (recommended) vs global constant

**Architect must adjudicate.** Boot prompt §1 Surface 1 recommends 100_000 μC global = 10% of common-agent (Agent_0..9) preseed 1.0 Coin. But G3.1 SHIPPED `classify_solvency` at `src/runtime/agent_pnl.rs:311` uses `initial_balance_micro / 10` per-agent — yielding:

| Agent | Preseed (μC) | 10% cap (μC) |
|-------|--------------|--------------|
| Agent_0..9 | 1_000_000 | 100_000 |
| MarketMakerBudget | 5_000_000 | 500_000 |
| tb7-7-sponsor | 10_000_000 | 1_000_000 |

**Recommended**: per-agent-keyed via `bankruptcy_risk_cap_micro(agent_id, q) -> i64 = initial_balance_micro_from_default_preseed(agent_id) / 10`. This is internally consistent with G3.1 (the read-only classifier and the admission predicate use the same threshold), and it matches the architect's "risk-cap floor" intent for high-balance system agents like MarketMakerBudget (10% of 5M = 500k floor is more risk-rule-meaningful than a flat 100k for an agent with 5M preseed).

**Alternative**: global `BANKRUPTCY_RISK_CAP_MICRO: i64 = 100_000` constant per boot prompt — simpler, lower test surface, but treats system agents identically to user agents and **diverges from G3.1 `NearInsolvent` classifier** (architect would have to ratify the divergence).

**Halt-and-re-charter trigger** (boot prompt §69): if architect picks per-agent **with a SEPARATE per-agent table** (NOT derived from preseed), that's a new `EconomicState.bankruptcy_risk_cap_t` sub-field → Class-4 schema extension → 2-packet split. Surface explicitly.

### Q2 — Gap-A reputation accumulation: verdict-uniform vs verdict-weighted

**Architect must adjudicate.** OBS_G2P_VERIFY_PEER_REWARD §2.1 documents the gap as "no `reputations_t` mutation in any sequencer arm." G3.2 closure choices:

- **Option A.uniform**: `+1` per accepted VerifyTx regardless of verdict (Confirm or Doubt). Simplest; doesn't bias verifiers toward Confirm. Easy to revert if architect later reclassifies.
- **Option A.confirm-weighted**: `+1` for Confirm, `+0` for Doubt (or `-1`). Models a bull-bias for verifier reputation.
- **Option A.outcome-correlated**: `+1` if the verifier's verdict matches the OMEGA-Confirm settlement; `-1` otherwise. Requires deferred reputation update (after the target's settlement) → state-tracking complexity → architect-§10 surface.

**Recommended**: Option A.uniform for G3.2 (closes the gap mechanically with minimal new state-tracking; verdict-weighting can land as a forward TB if PnL trajectory shows the bull-bias is harmful).
