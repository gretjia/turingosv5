## §2 Annotation layer (architect-original-text vs derivative-implementation)

### §2.1 Why path C → B′ over Option A

The AI-coder proposed three options for responding to Codex R1 VETO:

Architect rejected A explicitly: "Codex VETO is valid. The ID namespace

### §2.2 Canonical namespace decision (ruling §3)

The architect's §3 settles a long-standing latent ambiguity in the
v4 codebase: there are TWO id namespaces in flight — the **canonical**
(`WorkTx.tx_id`, used by Sequencer + EconomicState) and the **shadow**
(`tx_{count}_by_{author}`, generated by `bus.append` for the in-memory
`kernel.tape`). Pre-Atom-6 the legacy CPMM read-view operated entirely
in the shadow namespace (`kernel.markets[shadow_id]`,
`boltzmann_select_parent(&snap.tape, ...)`); the canonical chain ran
in parallel via `bus.submit_typed_tx` → Sequencer → L4. Codex's VETO
exposed that Atom 6's wire-swap mixed the two namespaces (price_index
canonical-keyed, mask_set computed against shadow Tape, evaluator
unwrapping canonical TxId into shadow append).

The ruling's canonical namespace decision settles this:

  - **PriceIndex** + **NodeMarketEntry** are keyed by canonical
    accepted `WorkTx.tx_id`. (Already true at HEAD `48e84ee`.)
  - **Shadow tape ids are legacy/local only**. The shadow
    `kernel.tape` continues to exist for prompt-context view sync but
    has no canonical authority.
  - **`compute_mask_set` MUST NOT read `kernel.tape`**. It must
    operate on canonical-keyed parent-child edges (CanonicalNodeGraph).
  - **Boltzmann selector output MUST NOT be consumed as legacy
    `bus.append` parent_id**. (Closed by B′ step 1 at commit `48e84ee`.)

### §2.3 CanonicalNodeGraph implementation contract (ruling §4)

The ruling specifies the data sources:

  - L4 accepted WorkTx (the canonical ledger of accepted proposals).
  - `ProposalTelemetry.parent_tx` (CAS-resident; linked from
    WorkTx.proposal_cid; carries the proposal-DAG parent edge).
  - challenge status (the `EconomicState.challenge_cases_t` index).

The graph is keyed by canonical TxId. The `compute_mask_set` signature
changes from `(econ, &Tape, policy, &price_index)` to `(econ,
&CanonicalNodeGraph, policy, &price_index)`. Halt-trigger #3 + the
existing `tests/tb_14_mask_set.rs` ship-gate tests are rewritten to
build `CanonicalNodeGraph` instead of `Tape`.

Open architectural questions for B′ step 4 implementation
(self-resolvable; if blocked, escalate):

  Q4.1 — Graph build location. Three options:
    (α) `bus.snapshot()` walks L4 + reads CAS each call. Expensive
        O(N + N·CAS_read) per snapshot.
    (β) `EconomicState` carries `canonical_edges_t: BTreeMap<TxId,
        BTreeSet<TxId>>` updated by Sequencer on WorkTx accept.
        Cheap O(1) at snapshot time. Schema change.
    (γ) Cached at `Sequencer` level (Arc<RwLock<Map>>); rebuilt on
        ledger writes. Schema unchanged; lock-coordination cost.
    Resolution: AI-coder chooses based on call frequency + atomicity.

  Q4.2 — CAS read coupling. CanonicalNodeGraph build needs CAS access
    if parent_tx is read from ProposalTelemetry. Sequencer's
    dispatch_transition is currently CAS-clean (pure over &QState +
    &TypedTx). Adding CAS coupling to dispatch breaks halt-trigger
    #2 spirit (sequencer dispatch must remain price-blind +
    canonical-only).
    Resolution proposal: read parent_tx at sequencer-level (where CAS
    access already exists for WorkTx.proposal_cid → ProposalTelemetry
    fetch) and store the edge in `EconomicState.canonical_edges_t`
    via a side-channel update during apply_one. The dispatch arm
    itself stays CAS-clean; the apply_one wrapper performs the CAS
    read. This satisfies both Q4.1 (β-style) and Q4.2.

### §2.4 Production smokes contract (ruling §5 + §6)

The ruling's §5 + §6 specify the WITNESS that B′ step 4 must produce
to evidence canonical masking actually works in production:

  Positive (§5):
    - parent accepted WorkTx A (real signed WorkTx, accepted by L4).
    - child accepted WorkTx B with parent_tx = A.
    - child price (from compute_price_index over EconomicState)
      dominates parent price by `policy.price_margin`.
    - liquidity sufficient (≥ `policy.min_liquidity`).
    - no unresolved challenge against B.
    - assert `mask_set` (from compute_mask_set on the new
      CanonicalNodeGraph) contains A.
    - assert `ChainTape` (canonical, not shadow) still contains A
      (CR-14.3 preservation across mask).

  Negatives (§6):
    - low-liquidity child cannot mask parent (CR-14.4 / SG-14.8).
    - unresolved-challenged child cannot mask parent (CR-14.5 /
      SG-14.7 / halt-trigger #6).
    - predicate-failed child cannot mask parent (CR-14.1; predicate
      gate at sequencer.rs:516-558 must reject failing children
      from L4, so they don't appear in canonical edges to begin
      with — this smoke verifies the structural decoupling).
