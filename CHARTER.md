# TuringOS V5 Charter

TuringOS V5 is the clean product repo bootstrapped from TuringOS V4 main.

V4 remains the development constitutional harness and evidence rail. V5 is the
future product world line. V5 runtime must not treat V4 handover/evidence,
genesis, local paths, dashboards, reports, task broadcast files, or MiniF2F
development corpora as runtime truth.

## Mission

Build an evidence-backed product path:

```text
/build -> spec turn -> generate -> ArtifactManifest -> preview -> modify -> regenerate -> download
```

All important state transitions must be reconstructable through accepted code,
contracts, evidence capsules, CI, review, and PR history.

## Initial Mode

Current phase is V5-R0 Clean Bootstrap.

This phase creates the repo, shared agent entry, harness docs, Task Broadcast
control plane, PR/CI scaffolding, and first harness gates. It does not ship
product features.

## Non-Claims

- V5 production ChainTape/genesis is not initialized yet.
- V4 evidence is development evidence only.
- Existing migrated v4 source is a starting archive, not a final V5 runtime
  contract.
- MiniF2F is not part of the V5 product core.
