# PR Policy

PRs are candidate state transitions.

Workers must open PRs from assigned branches. Workers do not merge PRs. Meta
reviews CI, diff, WorkerReport, audit, and Veto status before deciding.

PR title examples:

```text
[CLAIM][V5-C0-CONTRACT-ARTIFACT-001][Class2] Define ArtifactManifest contract
[V5-C0][Class2][Contract] Define ArtifactManifest contract
[REPAIR][V5-REPAIR-V5-C0-CONTRACT-ARTIFACT-001-001][Class2] Fix invariant
```
